Polimi Board Game Manager

Welcome!
This is a RESTful API created for the project of Middleware course of "Politecnico di Milano" in the year 2016-2017.
The API is accessible under https://polimi-board-game-manager.herokuapp.com/api/.
The source code is public and can be found at https://github.com/costi31/polimi-board-game-manager.

The requests relative to users that you can send to https://polimi-board-game-manager.herokuapp.com/api/users/, are:

  • POST https://polimi-board-game-manager.herokuapp.com/api/users/login :
    to login with "username" and "password" parameters sent in a form. A JSON Web Token is sent as authorization bearer in the response header.

  • POST https://polimi-board-game-manager.herokuapp.com/api/users/ :
    to create a new user with "fullName", "username" and "password" parameters sent in a form.

  • GET https://polimi-board-game-manager.herokuapp.com/api/users/ :
    to find all the users existing in the database, sorted and filtered with the optional criteria, that can be specified by sending multiple query parameters "filter=<filter_name>@<filter_value>" and "order=<order_by_field>@<order_mode>" (where order_mode can be asc for ascendent or desc for descendent).

  • GET https://polimi-board-game-manager.herokuapp.com/api/users/count :
    to get the count of all the existing users.

  • GET https://polimi-board-game-manager.herokuapp.com/api/users/<user_id> :
    to get the information relative to the user with the desired id=user_id.

  • PUT https://polimi-board-game-manager.herokuapp.com/api/users/<user_id> :
    to update the information relative to the user with the desired id=user_id.
    The information are sent as form parameters. If the user is found and is authenticated then he's updated. If the user_id is not found then a new user is created with that id.

  • DELETE https://polimi-board-game-manager.herokuapp.com/api/users/<user_id> :
    to delete the user with id=user_id.
    The user performing this request must be authenticated.

The requests relative to board games that you can send to https://polimi-board-game-manager.herokuapp.com/api/boardgames/, are:

  • POST https://polimi-board-game-manager.herokuapp.com/api/boardgames/ :
    to create a new board games with information sent in a form or as xml or json.

  • GET https://polimi-board-game-manager.herokuapp.com/api/boardgames/ :
    to find all the board games existing in the database, sorted and filtered with the optional criteria, that can be specified by sending multiple query parameters "filter=<filter_name>@<filter_value>" and "order=<order_by_field>@<order_mode>" (where order_mode can be asc for ascendent or desc for descendent).

  • GET https://polimi-board-game-manager.herokuapp.com/api/boardgames/count :
    to get the count of all the existing board games.

  • GET https://polimi-board-game-manager.herokuapp.com/api/boardgames/<boardgame_id> :
    to get the information relative to the board game with the desired id=boardgame_id.

  • PUT https://polimi-board-game-manager.herokuapp.com/api/boardgames/<boardgame_id> :
    to update the information relative to the board game with the desired id=boardgame_id.
    The information are sent as xml or json. If the board game is found then it's updated. If the boardgame_id is not found then a new board game is created with that id.
    The user performing this request must be authenticated and must be a super user.

  • DELETE https://polimi-board-game-manager.herokuapp.com/api/boardgames/<boardgame_id> :
    to delete the board game with id=boardgame_id.
    The user performing this request must be authenticated and must be a super user.

The requests relative to plays that you can send to https://polimi-board-game-manager.herokuapp.com/api/users/<user_id>/plays/ (for plays relative to user with id=user_id), are:

  • POST https://polimi-board-game-manager.herokuapp.com/api/users/<user_id>/plays/ :
    to create a new play relative to user with id=user_id with information sent as xml or json.
    The user performing this request must be authenticated and must correspond to the one relative to the play.

  • GET https://polimi-board-game-manager.herokuapp.com/api/users/<user_id>/plays/ :
    to find all the plays existing in the database relative to the user with id=user_id, sorted and filtered with the optional criteria, that can be specified by sending multiple query parameters "filter=<filter_name>@<filter_value>" and "order=<order_by_field>@<order_mode>" (where order_mode can be asc for ascendent or desc for descendent).

  • GET https://polimi-board-game-manager.herokuapp.com/api/users/<user_id>/plays/<play_id> :
    to get the information relative to the play with the desired id=play_id.

  • PUT https://polimi-board-game-manager.herokuapp.com/api/users/<user_id>/plays/<play_id> :
    to update the information relative to the play with the desired id=play_id.
    The information are sent as xml or json. If the play is found then it's updated. If the play_id is not found then a new play is created with that id.
    The user performing this request must be authenticated and must correspond to the one relative to the play.

  • DELETE https://polimi-board-game-manager.herokuapp.com/api/users/<user_id>/plays/<play_id> :
    to delete the play with id=play_id.
    The user performing this request must be authenticated and must correspond to the one relative to the play.
